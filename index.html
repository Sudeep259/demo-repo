Git & GitHub — Complete Command Cheat Sheet (A4 Printable)

Commands, short examples and when to use them. Real-world scenarios + step-by-step resolutions included. Save this page as PDF for offline study.

Setup & Configuration

git --versionCheck Git is installed. git --version
git config --global user.nameSet author name. git config --global user.name "Sudeep Kumar"
git config --global user.emailSet author email. git config --global user.email "you@domain.com"
git config core.editorSet commit editor. git config core.editor "code --wait"
git help <cmd>Open help page. git help commit

Create / Clone Repositories

git initCreate new local repo. git init
git clone <url>Clone remote repo locally. git clone https://github.com/you/repo.git
git clone --depth 1 <url>Shallow clone (no full history). git clone --depth 1 ...
git remote -vShow configured remotes.
git remote add origin <url>Add remote. git remote add origin https://github.com/you/repo.git
git remote set-url origin <url>Change remote URL. git remote set-url origin git@github.com:you/repo.git

Staging & Committing

git statusShow working tree status. git status
git add <file>Stage a file. git add index.html
git add .Stage all changes. git add .
git restore --staged <file>Unstage a file. git restore --staged index.html
git commit -m "msg"Commit staged changes. git commit -m "Initial commit"
git commit -am "msg"Stage tracked files & commit. git commit -am "Update README"
git commit --amendAmend last commit (before push). git commit --amend -m "Fix typo"
git log --oneline --graph --decorate --allCompact visual history.

Branching

git branchList local branches. git branch
git branch -aList all branches (including remotes).
git checkout -b <name>Create & switch to new branch. git checkout -b feature/login
git switch <name>Switch branch (modern). git switch main
git branch -d <name>Delete local branch (safe). git branch -d feature/x
git branch -D <name>Force delete local branch.
git merge <branch>Merge branch into current. git merge feature/login
git rebase <base>Reapply commits onto base (linear history). git rebase main
git rebase -i HEAD~nInteractive rebase to squash/reorder commits.
git cherry-pick <commit>Apply a single commit on current branch. git cherry-pick a1b2c3

Remote Operations (Push / Pull / Fetch)

git fetchDownload remote updates (no merge). git fetch origin
git pullFetch + merge. git pull
git pull --rebaseFetch & rebase local commits onto remote. git pull --rebase origin main
git pushPush current branch to upstream. git push origin main
git push -u origin <branch>Push & set upstream. git push -u origin feature/login
git push --force-with-leaseSafer force-push: fail if remote changed unexpectedly.
git push origin --delete <branch>Delete remote branch.
git fetch --all --pruneFetch all remotes & remove deleted refs.

Viewing Diffs & History

git diffShow unstaged changes.
git diff --stagedShow staged changes.
git diff branch1..branch2Diff between two branches. git diff main..feature/x
git show <commit>Show commit details and changes.
git blame <file>Show author for each line of a file.
git reflogView reference log (helpful to recover lost commits).

Undoing Changes & Recovery

git restore <file>Discard unstaged changes in working tree. git restore index.html
git restore --staged <file>Unstage a file. git restore --staged index.html
git reset --soft HEAD~1Undo last commit but keep changes staged.
git reset --mixed HEAD~1Undo last commit and keep changes unstaged.
git reset --hard HEAD~1Destructive: remove last commit and changes (local).
git revert <commit>Make new commit that undoes the given commit (safe for shared branches).
git clean -fdRemove untracked files and directories (careful!).

Stashing (temporary save)

git stashTemporarily save local changes. git stash
git stash listList stashes.
git stash applyReapply stash (keeps it).
git stash popReapply and remove top stash.
git stash push -m "msg"Save stash with message.

Tags & Releases

git tag -a v1.0 -m "msg"Create annotated tag. git tag -a v1.0 -m "Release v1.0"
git push origin v1.0Push a tag. git push origin v1.0
git push origin --tagsPush all tags.
git tag -d v1.0Delete local tag.
git push origin :refs/tags/v1.0Delete remote tag.

Advanced History Rewriting

git rebase -i HEAD~nInteractive rebase for squash/reorder/fix commits.
git commit --amendEdit last commit (before push).
git bisectBinary search for the commit that introduced a bug.
git filter-repoRewrite history (preferred to filter-branch) — use with care.

GitHub CLI (useful)

gh auth loginAuthenticate GitHub CLI.
gh repo createCreate a new GitHub repo and push local code.
gh pr createCreate PR from CLI. gh pr create --title "Add X" --body "desc"
gh pr checkout <id>Checkout a PR locally.
gh issue createCreate GitHub issue from CLI.

Real-World Scenarios & Step-by-Step Resolutions

Scenario A — Files nested in a subfolder on GitHub (you want files at root)

Cause: You initialized/pushed from inside parent folder so Git tracked the inner folder as top-level content.

Fix (recommended): Initialize Git inside the inner project folder so only its contents are tracked.

cd demo-repo/localRepo
git init
git remote add origin https://github.com/you/demo-repo.git
git add .
git commit -m "Add project files"
git branch -M main
git push -u origin main

Alternate: If .git must stay in parent, move files up, remove folder, commit the change and push.

Scenario B — Push rejected: non-fast-forward

Cause: Remote has commits you don’t have locally.

Safe fix:

git pull --rebase origin main
# resolve conflicts if any
git push origin main

If you intentionally want to overwrite remote (coordinate with team): git push --force-with-lease origin main.

Scenario C — Merge conflict while merging or rebasing

Steps to resolve:

git status                      # shows conflicted files
# open files and resolve <<<<<<< >>>>>>> markers
git add 
# if merge: git commit
# if rebase: git rebase --continue
git push origin your-branch

Use VS Code merge editor or git mergetool for guided resolution.

Scenario D — Sensitive file committed and pushed (API key)
  1. Rotate / revoke the exposed credentials immediately.
  2. Remove file and add to .gitignore: 
    git rm --cached secret.txt
echo "secret.txt" >> .gitignore
git commit -m "Remove secret and add to .gitignore"
git push origin main
  3. To remove from history (coordination required): use git filter-repo then force-push with lease.

Rewriting history affects all collaborators — communicate first.

Scenario E — Someone pushed directly to main and build broke

Fast revert and hotfix:

git checkout main
git pull origin main
git revert <bad-commit-hash>
git push origin main
# create hotfix branch if needed
git checkout -b hotfix/fix-build
# fix, commit, open PR

Protect main branch with branch rules to prevent direct pushes.

Scenario F — Recover lost commit after reset 
git reflog
# find commit hash (abc123)
git checkout -b recovered abc123

Reflog shows local movement, even after destructive commands.

Scenario G — Switch branches but have uncommitted work 
git stash push -m "WIP: work"
git switch other-branch
# later
git switch feature-branch
git stash pop

Or commit to a WIP branch to preserve history.

Scenario H — Cherry-pick bug fix into release branch 
git log                      # find commit hash a1b2c3 on feature branch
git checkout release/1.0
git cherry-pick a1b2c3
git push origin release/1.0
Scenario I — Squash multiple commits before merging 
git checkout feature/x
git rebase -i HEAD~5
# mark commits as 's' (squash) or 'f' (fixup)
git push --force-with-lease origin feature/x

Coordinate with team before force-pushing rebased branches.

Interview Focus: Common Commands to Practice

Memorize and be ready to demonstrate these live:

  • git init, git clone <url>, git add ., git commit -m "msg"
  • git checkout -b <branch>, git pull --rebase origin main, git push -u origin <branch>
  • git merge <branch> vs git rebase <base>, git revert <commit>
  • git stash, git reflog, git reset --hard <commit> (know risks)

Best Practices to Mention in Interviews

  • Use feature branches for each task; open PRs for reviews and CI checks.
  • Keep commits small and descriptive (imperative style).
  • Protect main with branch protection rules & require passing CI checks.
  • Prefer git revert for undoing public commits; use --force-with-lease when rewriting history.
  • Always .gitignore generated files and never commit secrets (rotate keys immediately if leaked).
